# Glossaire IT — acronymes et termes

## A

- **ACL — Access Control List** : liste de règles d'accès à une ressource ou un flux.
- **AD DS — Active Directory Domain Services** : annuaire de domaine Microsoft local.
- **ADUC — Active Directory Users and Computers** : console de gestion des objets AD.
- **APIPA — Automatic Private IP Addressing** : adresse `169.254.x.x` autoattribuée en cas d'échec DHCP.
- **ARP — Address Resolution Protocol** : associe une IPv4 locale à une adresse MAC.

## B

- **BIOS/UEFI** : micrologiciel qui initialise le matériel et démarre l'OS.
- **BitLocker** : chiffrement de volume Windows.
- **BSOD** : écran bleu après erreur critique du noyau Windows.

## C

- **CIDR** : notation de préfixe IP, par exemple `/24`.
- **Conditional Access** : politiques cloud qui évaluent des signaux avant d'autoriser l'accès.
- **CPU** : processeur.

## D

- **DC — Domain Controller** : contrôleur de domaine.
- **DHCP** : attribue automatiquement IP, masque, passerelle et DNS.
- **DNS** : traduit les noms en adresses et publie des informations de service.
- **DORA** : Discover, Offer, Request, Acknowledge, échange DHCP initial.

## E

- **EDR** : détection et réponse sur les appareils.
- **Entra ID** : service d'identité cloud Microsoft.
- **ESP** : protocole IPsec qui protège le trafic.

## F

- **FQDN** : nom DNS complet, par exemple `server01.corp.example`.
- **FTP/SFTP** : transfert de fichiers; SFTP passe par SSH et n'est pas FTP chiffré.

## G

- **Gateway** : routeur utilisé pour joindre d'autres réseaux.
- **GPO** : objet de stratégie de groupe.
- **GPU** : processeur graphique.

## H

- **Hash** : empreinte calculée d'un fichier ou contenu.
- **HTTP/HTTPS** : protocole web; HTTPS utilise TLS.

## I

- **IAM** : gestion des identités et des accès.
- **ICMP** : messages réseau, notamment utilisés par ping.
- **IMAP** : accès aux courriels conservés sur le serveur.
- **IP** : protocole d'adressage et routage.
- **IPsec** : suite de protocoles de sécurité réseau.

## K

- **Kerberos** : protocole d'authentification à tickets utilisé dans AD.

## L

- **LAN** : réseau local.
- **LDAP/LDAPS** : protocole d'accès à un annuaire; LDAPS est protégé par TLS.
- **Least privilege** : donner uniquement les droits nécessaires.

## M

- **MAC address** : identifiant de couche liaison d'une interface.
- **MDM** : gestion des appareils mobiles et postes.
- **MFA** : authentification multifacteur.
- **MTU** : taille maximale d'un paquet sur une interface avant fragmentation ou adaptation.

## N

- **NAT** : traduction d'adresses réseau.
- **NLA** : authentification au niveau réseau pour RDP.
- **NTFS** : système de fichiers Windows avec permissions avancées.
- **NTP** : synchronisation de l'heure.

## O

- **OAuth 2.0** : cadre d'autorisation moderne.
- **OIDC — OpenID Connect** : couche d'identité sur OAuth 2.0.
- **OS** : système d'exploitation.
- **OU** : unité organisationnelle dans AD.

## P

- **Patch** : correctif logiciel.
- **Ping** : test ICMP de portée et latence, si autorisé.
- **POP3** : protocole de récupération de courriel.
- **Port** : numéro logique associé à un service TCP ou UDP.
- **PST/OST** : fichiers de données Outlook; PST peut contenir des données locales, OST est souvent un cache.

## R

- **RAID** : combinaison de disques pour performance ou disponibilité; ce n'est pas une sauvegarde.
- **RAM** : mémoire vive.
- **RDP** : protocole de bureau à distance Windows.
- **RMM** : outil de gestion et surveillance à distance.
- **RPO/RTO** : objectifs de perte de données et de temps de restauration.

## S

- **SaaS** : logiciel fourni comme service cloud.
- **SAML** : protocole d'échange d'assertions d'identité.
- **SLA** : engagement de niveau de service.
- **SMB** : partage de fichiers et imprimantes Windows.
- **SMTP** : transport et soumission de courriels.
- **SNMP** : supervision d'équipements.
- **SSH** : accès distant sécurisé en ligne de commande.
- **SSID** : nom d'un réseau Wi-Fi.
- **Subnet** : sous-réseau IP.

## T

- **TCP** : transport fiable orienté connexion.
- **TLS** : protection cryptographique des communications.
- **TPM** : module matériel de sécurité utilisé notamment par BitLocker.

## U

- **UDP** : transport sans connexion, rapide mais sans garantie de livraison.
- **UPN** : nom de connexion de type utilisateur@domaine dans AD.

## V

- **VLAN** : segmentation logique d'un réseau de couche 2.
- **VNC** : protocole/outil de partage d'écran.
- **VPN** : tunnel privé chiffré.

## W

- **WAN** : réseau étendu.
- **Wi-Fi** : réseau local sans fil IEEE 802.11.
- **WSUS** : service Microsoft de gestion de mises à jour Windows en environnement local.

## Z

- **Zero Trust** : approche qui vérifie explicitement l'identité, l'appareil, le contexte et limite les privilèges au lieu de faire confiance au réseau par défaut.

## Méthode d'apprentissage

Pour chaque terme, sois capable de répondre à trois questions :

1. À quoi sert-il?
2. Dans quel scénario de soutien le rencontres-tu?
3. Quel outil ou test permet de le vérifier?
